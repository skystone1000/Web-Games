## Game Input

```
GAME_TITLE:       2048
GAME_FOLDER:      2048
GAME_EMOJI:       🔢
GENRE_TAG:        Puzzle

SHORT_HUB_DESC:   Slide numbered tiles across a 4×4 grid and merge matching values to reach 2048.
                  Simple to learn, endlessly satisfying to master.

LAYOUT_TYPE:      B — Two-column panels
                  Left panel: 4×4 tile grid + score bar above it.
                  Right sidebar: title, description, stat counters, status pill, rules, action buttons.

MOBILE_SCROLL:    no

GAME_DESCRIPTION:
  Objective: Merge tiles by sliding the entire board in one of four directions (up, down, left, right).
  When two tiles with the same number collide they merge into one tile worth their sum. Keep merging
  to reach the 2048 tile — or push for the highest score possible.

  Win condition: Reaching a tile with the value 2048 triggers a win overlay ("You reached 2048!").
  The player may choose to keep playing for a higher score. Game over when the board is full and no
  adjacent tiles can merge.

  Turn structure: Turn-based (each swipe/keypress = one turn). After every move a new tile (value 2
  with 90% probability, value 4 with 10%) spawns in a random empty cell.

  Player count: Single player.

  Special mechanics:
  - All tiles in the chosen direction slide as far as possible before merging.
  - Each pair merges at most once per move (a 4-4-4-4 row becomes 8-8, not 16).
  - Newly spawned tile does a brief scale-in animation; merged tile does a brief pop animation.
  - Tile background colour scales with value: 2 (surface2), 4, 8, 16, 32, 64 (orange tones),
    128, 256, 512 (yellow), 1024, 2048 (primary/accent glow).

  Score model:
  - Score increments by the value of every merged tile (merging two 512s = +1024).
  - Best score persists via localStorage key "2048Best".

  Visual feedback:
  - Per-tile colour palette (2 → 2048) — bright saturated colours matching the dark theme.
  - Merge pop: `transform: scale(1.15)` → `scale(1)` over 120ms.
  - Spawn scale-in: `scale(0)` → `scale(1)` over 150ms.
  - Win overlay with animated confetti emoji or CSS shimmer on the 2048 tile.

CONTROLS:
  - Arrow keys / WASD: slide board in that direction
  - Space: not used (no pause state — game is always live between moves)
  - Swipe left/right/up/down: slide board (mobile)
  - "New Game" button: reset board and score
  - "Keep Going" button (win overlay): dismiss win and continue

SIDEBAR_STATS:
  - Score (resets each game)
  - Best (persists via localStorage "2048Best")
  - Moves (total slides taken this game)

EXTRA_REQUIREMENTS:
  - Tile colour map (background / text) to define in CSS variables or inline:
      2 → bg #1A2438, text #E8EDF5
      4 → bg #1E2B45, text #E8EDF5
      8 → bg #C06000, text #fff
      16 → bg #E07000, text #fff
      32 → bg #E05030, text #fff
      64 → bg #E03010, text #fff
      128 → bg #D4A800, text #fff
      256 → bg #DDBB00, text #fff
      512 → bg #E8CC00, text #fff
      1024 → bg var(--primary), text #fff
      2048 → bg var(--accent), text var(--bg)
  - Board must be a CSS grid (not canvas); tiles are absolutely positioned divs inside a
    relative container for smooth CSS translate animations.
  - No keyboard shortcuts other than arrow/WASD. Do not capture Space.
```
